On peut voir sur l\textquotesingle{}image que la ESP8266 s\textquotesingle{}est bien connectée sur la Wifi Station via l\textquotesingle{}IP du routeur sur la wifi \char`\"{}robotique\char`\"{}. On peut aussi constater que la carte a configuré comme il faut les paramètres Access Point qui par ailleurs est bien visible dans la recherche de connexion Wifi \+:

 

Tout ceci est afficher via la connexion au port Serial COM et de plus on remarque la connexion au serveur \mbox{\hyperlink{class_m_q_t_t}{MQTT}} en tant que USER9.\hypertarget{index_autotoc_md5}{}\doxysubsection{Communication sur le serveur MQTT}\label{index_autotoc_md5}


On peut voir l\textquotesingle{}affichage sur le port Serial les valeurs de température et d\textquotesingle{}humidité (arrondis à 2 décimales) qui sont envoyés par intervalle de 2 secondes sur les 2 topics de paramètre \+: Temp et Humi

Ils seront ensuite récupérés par le Subscriber pour l\textquotesingle{}ouverture/fermeture du relais.\hypertarget{index_autotoc_md6}{}\doxysubsection{Affichage sur le serveur Web}\label{index_autotoc_md6}


Notre code affiche bien sur l\textquotesingle{}adresse IP la page Web qui nous indique les IP de notre configuration \mbox{\hyperlink{class_w_s}{WS}} et AP. Il nous donne aussi la possibilité de regarder les paramètres envoyés par le capteur de Température et d\textquotesingle{}Humidité \+:



La page web nous affiche les différents paramètres avec l\textquotesingle{}unité et la page Web sera rechargée toutes les 2 secondes (correspondant aussi au rechargement pour l\textquotesingle{}envoie des paramètres sur le serveur \mbox{\hyperlink{class_m_q_t_t}{MQTT}}) afin de toujours avoir des valeurs actuelles.
